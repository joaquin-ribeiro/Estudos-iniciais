Atualmente podemos declarar variaveis de 3 manieras diferentes, sendo elas:


Var
    Foi utilizada por muito tempo como a unica maneira de declarar variaveis no js, hj em dia já não é mais usada por ter se tornado obsoleta em relação as novas formas;
    Possui caracteristicas de escopo global, podendo ser chamada de qualquer lugar do codigo, por isso seu nome tem que ser unico pra não gerar conflitos;
    OBS: Seu valor pode ser alterado a qualquer momento e sofre hosting.

    Ex: var idade = 22 

Let 
    A nova forma padrão para se declarar variaveis mutaveis no js desde 2015, possui o escopo de bloco (seu nome seu valor se limitar a um bloco de codigo, portanto pode se existir diversos lets com o mesmo nome de variavel desde que estejam em blocos diferentes)
    OBS: Sobre hosting porem apenas na zona morta, pode ser reatribuida (let idade = 22 // idade = 23, mas n redeclarada let idade = 23)

    Ex: let idade = 22

Const
    É praticamente um let com uma restrição a mais, uma const não pode ter seu valor redefinido ou redeclarado, se ela começa o codigo com valor x, termina com o mesmo valor x, até o final dele.
    OBS: sofre hosting apenas em zona morta, possui escopo de bloco, sendo limitado por codigo de ambos os lados {}

    Ex: const idade = 22


HOISTING

    O conceito de hoisting é em suma 'içar' determinados itens antes do começo do codigo, variaveis são puxadas junto com as funções
    EX: console.log(a)
        var a = 'bom dia'
        Aqui o console vai retornar o valor de a como undefined, pois ele lê essa linha de codigo como
        var a (o js cria a variavel que vai ser usada mas n determina um valor pra ele deixanmdo do tipo undefined)
        console.log(a) // undefined
        var a = 'bom dia'

        Com o var acontece de retornar o valor de undefined já com o let e o const as variaveis são içadas mas entram em zona morta portanto dá erro de referenceError no console.

    Porém com funções não ocorre a mesma coisa, já que elas são puxadas inteiras antes de onde estão sendo chamadas.

    Ex: fala()
        function fala(){
            console.log('oi')
        }    
        Mesmo chamando a função antes de declarar ela, ainda vai dar certo pois aqui ela iça a função antes de rodar o codigo.


REGRAS DE NOMENCLATURA

    No js temos algumas regras ao declararmos variaveis sendo elas:
    Uma variavel pode começar com letra, $ ou _
    let idade  = 22
    let $idade = 22
    let _idade = 22
    
    pode conter numeros mas n pode começar com eles
    let idade2 = 23 OK
    let 2idade = 23 NOT OK
    
    pode usar letras maiusculas
    let idade = 22   || let Idade = 22
    2 variaveis diferentes pelo maisculo na frente (case sensitive)

    Não se pode usar espaços
    let dia de hoje = yy/yy/yyyy  NOT OK
    let diadehoje = yy/yy/yyyy 
    nesse caso seria melhor usar diadehoje

    Não pode usar palavras reservadas para a linguagem como

    let let = 22
    let if = 32 

    A melhor maneira de chamar uma variavel seria 
    começar com minuscula e se existir uma segunda palavra usar maiuscula
    let minhaIdade = 89
    let mesAtual = 8
